# payment-service 故障处理手册

## 常见告警

- CPU 连续 5 分钟高于 90%
- 错误率高于 3%
- 接口平均耗时高于 800ms

## 排查建议

1. 先检查最近 30 分钟的流量变化和最近一次发布记录。
2. 关注 Prometheus 中 CPU、错误率、P95 延迟是否同时抬升。
3. 查询 error 和 timeout 日志，观察是否存在下游依赖放大。
4. 检查最近慢 SQL，如果 rows 或 duration 明显上升，优先定位数据库热点查询。
5. 短期止损优先考虑限流、熔断和回滚最近变更。

## 经验总结

payment-service 在大促期间更容易因慢 SQL 叠加流量高峰触发 CPU 和错误率告警。CPU 抬升通常不单独出现，往往伴随 timeout、重试和数据库压力上升。
