# AegisFlow 接口文档

本文档对应新的 `/api/v2` Agent 平台接口，真实服务端实现已经切换到 `session / run / event / resume / knowledge / tools` 模型。

## 接口总览

| 场景 | 方法 | 路径 | 说明 |
| --- | --- | --- | --- |
| 创建会话 | `POST` | `/api/v2/sessions` | 创建 chat 或 ops 会话 |
| 聊天运行 | `POST` | `/api/v2/runs/chat` | 执行一次 Chat ReAct 运行 |
| 流式聊天运行 | `POST` | `/api/v2/runs/chat/stream` | 通过 SSE 返回 run_event 和 done |
| 运维运行 | `POST` | `/api/v2/runs/ops` | 执行一次 Supervisor + PlanExecute 运维工作流 |
| 查询运行 | `GET` | `/api/v2/runs/{runId}` | 查询单次运行详情 |
| 查询事件 | `GET` | `/api/v2/runs/{runId}/events` | 查询持久化事件时间线 |
| 恢复运行 | `POST` | `/api/v2/runs/{runId}/resume` | 对 interrupt run 进行审批恢复 |
| 上传知识文档 | `POST` | `/api/v2/knowledge/documents` | 上传知识文档 |
| 创建索引任务 | `POST` | `/api/v2/knowledge/index-jobs` | 创建异步知识索引任务 |
| MCP 工具目录 | `GET` | `/api/v2/tools/mcp` | 查询 API 当前发现到的 MCP 工具 |

## 1. 创建会话

`POST /api/v2/sessions`

```json
{
  "title": "Chat Agent Session",
  "mode": "chat"
}
```

## 2. Chat Run

`POST /api/v2/runs/chat`

```json
{
  "sessionId": "sess-001",
  "query": "支付服务 CPU 告警应该怎么排查？",
  "topK": 4
}
```

## 3. Chat Run SSE

`POST /api/v2/runs/chat/stream`

SSE 事件：

- `run_event`：单条运行事件
- `done`：最终 run 对象
- `error`：异常信息

## 4. Ops Run

`POST /api/v2/runs/ops`

```json
{
  "sessionId": "sess-ops-001",
  "alertTitle": "payment-service cpu usage high",
  "serviceName": "payment-service",
  "severity": "critical",
  "summary": "支付服务 CPU 连续 5 分钟高于 90%，错误率上升。"
}
```

## 5. Resume Run

`POST /api/v2/runs/{runId}/resume`

```json
{
  "interruptId": "agent:ReporterAgent;tool:approval.request",
  "approved": true,
  "note": "Approved from demo console"
}
```

## 6. 知识文档上传

`POST /api/v2/knowledge/documents`

请求类型：`multipart/form-data`

字段名：`file`

## 7. 知识索引任务

`POST /api/v2/knowledge/index-jobs`

```json
{
  "documentIds": ["kdoc-001"],
  "chunkSize": 500,
  "overlap": 80,
  "topK": 4
}
```

## 8. MCP 工具目录

`GET /api/v2/tools/mcp`

返回 API 当前通过 `mcp-go client + Eino MCP wrapper` 发现到的工具列表，例如：

- `prometheus.query`
- `logs.search`
- `mysql.query`
- `runbook.search`
